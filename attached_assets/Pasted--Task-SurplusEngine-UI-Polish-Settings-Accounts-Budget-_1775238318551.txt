# Task: SurplusEngine UI Polish — Settings, Accounts, Budget & Transactions

## Context
SurplusEngine is a personal finance app built with React 19 + TypeScript, dark theme, card-based layout with glassmorphism styling. This task covers UI/UX improvements across 4 pages. Do NOT change the color scheme, overall layout structure, or break existing data bindings/API integrations.

## Design Direction: Enhanced Glassmorphism
The app already uses glassmorphism on cards and surfaces. Enhance and make it more consistent across all pages:
- Ensure all cards use a consistent frosted glass effect: semi-transparent background (e.g., `rgba(255,255,255,0.04)` to `rgba(255,255,255,0.07)`), `backdrop-filter: blur(12px)`, and a subtle 1px border with `rgba(255,255,255,0.08)`
- Add layered depth: nested elements within glass cards (e.g., input fields, inner panels, table rows on hover) should use a slightly different glass layer — a touch more opaque or a tighter blur — to create a sense of depth-within-depth
- Hover states on interactive glass cards (credit cards, goal cards, budget items) should subtly increase the background opacity and border brightness for a "glow" effect
- Ensure glass surfaces have a very subtle inner shadow or gradient sheen (top-left to bottom-right, white at ~2% opacity) to reinforce the glass illusion
- Dropdowns, modals, and slide-out panels should also use the glass treatment — no opaque flat surfaces breaking the aesthetic
- Test that glassmorphism looks correct across all pages and doesn't cause readability issues with text contrast

# PAGE 1: SETTINGS

## 1.1 Billing Cycle & Currency Cards
- Both cards are functional but visually sparse — they have different content heights causing an unbalanced look
- Make both cards the same height with content vertically centered
- Add a subtle "saved" confirmation indicator (brief green checkmark or toast) when the user changes the start day or currency, so the user knows the change persisted

## 1.2 Category Manager — Grouping & Visual Hierarchy
- The category list is a flat, monotonous list where every row looks identical. Group categories visually by type:
  - Add section headers: "Expense Categories" and "Income Categories" with a subtle divider between them
  - Within each group, show a count badge (e.g., "Expense Categories (16)")
  - Remove the redundant "Expense" / "Income" badge from each individual row since the grouping already communicates type
- Alternatively, if keeping the flat list, at minimum sort so all Expense categories appear first, then Income, with a clear visual separator between them

## 1.3 Category Manager — Icon Consistency
- Some categories have meaningful icons (Food has a utensils icon, Transportation has a car icon) while many others use a generic tag icon (Spa, Drinks, Electricity, SIP, Loan EMI, Subscriptions, Shopping, etc.)
- Assign distinct, recognizable icons to every category:
  - Spa → spa/lotus icon
  - Drinks → coffee/glass icon
  - Electricity → lightning bolt icon
  - SIP → trending-up/chart icon
  - Loan EMI → bank/building icon
  - Subscriptions → refresh/repeat icon
  - Shopping → shopping-bag icon
  - Bonus → gift/star icon
  - Interest → percentage icon
  - Cashback → arrow-down-circle icon
  - Salary → wallet/briefcase icon
- Use icons from whatever icon library is already in use (likely lucide-react or similar)

## 1.4 Category Manager — Usability
- Add a search/filter input above the category list so users can quickly find a category when the list is long
- Add drag-to-reorder capability (or at minimum up/down arrow buttons) so users can control the display order of categories — this order should carry over to dropdowns and charts across the app
- The edit (pencil) and delete (trash) icons are very close together — add more horizontal spacing (at least 12px gap) to reduce mis-tap risk, especially on mobile
- When the user clicks edit, make the row transform into an inline edit state (name becomes an input, type becomes a dropdown) rather than opening a modal — keep the interaction lightweight

## 1.5 Category Manager — Add Category Form
- The "New category name" input, type dropdown, and "+ ADD" button are all on one line which is good, but the input field is disproportionately wide compared to the dropdown
- Make the input take ~60% width, dropdown ~20%, and button ~20% for better visual balance
- Add validation feedback: if the user tries to add a duplicate category name, show an inline error below the input rather than a toast or alert

# PAGE 2: MANAGE ACCOUNTS

## 2.1 Net Worth Summary Card
- The breakdown text ("Banks: ₹5,56,767.53  CC Outstanding: ₹59,163.26  Loans: ₹16,24,528.00") is just a plain text line — convert this into a horizontal segmented bar showing the proportion of assets vs liabilities visually
- Add a small trend indicator (arrow up/down + percentage) showing net worth change from last month

## 2.2 Credit Cards Section — Utilization Rings
- The circular utilization indicators (1%, 2%, 0%, 3%, 0%, 1%, 1%) are useful but very small and hard to read at a glance
- Make the rings slightly larger (increase from ~40px to ~56px diameter)
- Add color coding to the rings based on utilization thresholds:
  - 0-30% → green (healthy)
  - 30-50% → yellow (caution)
  - 50%+ → red (high utilization)
- Currently all rings look the same teal/blue color regardless of utilization level

## 2.3 Credit Cards — Due Date Visibility
- Due dates are shown in small light text ("Due 31th", "Due 11th", "Due 8th", "Due 21th", "Due 12th", "Due 4th") that's easy to miss
- Fix ordinal suffixes: "31th" should be "31st", "21th" should be "21st", "12th" is correct, "4th" is correct, "11th" is correct, "8th" is correct — audit and fix the ordinal suffix logic globally

## 2.4 Action Icons (Sync / Edit / Delete)
- The three action icons on each card (sync, edit, trash) are small and cramped together
- Increase touch targets — add more spacing between icons (at least 16px gap)
- Add tooltips on hover: "Sync Balance", "Edit Account", "Delete Account"
- Consider moving these into a "..." overflow menu to reduce visual clutter on each card, showing the actions on click

## 2.5 Loans Section
- The HDFC Loan card has good information (EMI, Rate, From account, repaid %) but the layout feels like a flat key-value dump
- The "18% repaid" progress bar is great — make it more prominent. Increase the bar height and add the percentage label inside or directly above the bar
- Add derived metrics: "Months remaining: X" and "Estimated payoff: MMM YYYY" below the progress bar
- The orange+blue segmented progress bar is interesting (likely showing principal vs interest) — add a legend explaining what each color segment represents

## 2.6 Section Headers
- Each section (Bank Accounts, Credit Cards, Loans) has a count badge and a total amount on the right with a collapse chevron — this is good
- Add a subtle bottom border or separator between sections for clearer visual grouping
- The total amounts (₹5,56,767.53 / ₹59,163.26 / ₹16,24,528.00) should use consistent coloring: green for assets (bank accounts), red for liabilities (CC outstanding, loans)

# PAGE 3: BUDGET ANALYSIS

## 3.1 Top Summary Cards
- The 5 summary cards (Cycle Progress, Overall, Net Difference, Fixed, Discretionary) are informative but only "Cycle Progress" has a progress bar — the others are just numbers
- Add mini progress bars to Fixed and Discretionary cards showing spent/budget ratio
- The "Net Difference" card showing "+₹38,406.48" in green is great — but this is confusing at first glance. Is this overspend or underspend? Add a tiny label: "Under budget" or "Over budget" to clarify direction

## 3.2 Fixed Commitments Section
- The 2-column grid layout with Planned/Actual/Difference per item is functional
- The "Paid" (green) and "Pending" (gray) status indicators are clear and useful
- Issue: "Pending" items show ₹0.00 as Actual and the full planned amount as Difference in red — this is technically correct but visually alarming for items that simply haven't been paid yet this cycle. Differentiate between "not yet paid" (show difference in yellow/amber as "pending") vs "overdue/missed" (show in red)
- The progress bars on each item should be color-coded:
  - Fully paid / under budget → green bar
  - Partially paid → yellow/amber bar
  - Overspent → red bar
  - Currently they all appear green regardless of status

## 3.3 Planned Amount Input Fields
- The planned amount inputs look editable inline which is great
- Add a visual indicator that these are editable (subtle pencil icon or a different background shade) — right now they look like static text until you click
- When the user changes a planned amount, show a brief save confirmation and recalculate the summary cards and difference columns immediately

## 3.4 Discretionary Section
- The section header shows "₹38,879.52 / ₹30,000.00" which means discretionary spending has EXCEEDED the budget — this should be visually alarming
- Color the total red and add a warning indicator (exclamation icon or a red banner) since the user is over budget on discretionary spending
- If the discretionary section has individual line items (like Fixed Commitments), show them in the same Planned/Actual/Difference format
- Add a "Top overspend categories" callout highlighting which discretionary categories are driving the overspend

## 3.5 Overall Page Structure
- Add a month/cycle navigation control at the top (left/right arrows to browse previous cycles) so users can compare budget performance across months
- Consider adding a mini bar chart showing budget adherence over the last 3-6 cycles to give trend context — "Are you getting better or worse at sticking to budget?"

# PAGE 4: TRANSACTIONS (LEDGER)

## 4.1 Table — Missing Edit Capability
- Each row only has a delete (trash) icon — there's no way to edit a transaction
- Add an edit icon (pencil) next to the delete icon on each row
- Clicking edit should either open an inline edit mode (preferred) or a slide-out panel with editable fields for date, description, category, account, and amount

## 4.2 Table — Missing Pagination / Load More
- The table currently shows ~15 rows with no indication of how many more exist
- Add pagination controls at the bottom (page numbers or "Load More" button)
- Show a total count: "Showing 1-15 of 247 transactions"

## 4.3 Filter Bar Improvements
- The search + category + time filter bar is good but could be enhanced:
  - Add an Account filter dropdown (filter by "One Card", "HDFC Salary", etc.)
  - Add an amount range filter (min/max) for finding specific transactions
  - Add a "Type" filter: All / Expense / Income / Transfer
- Show active filter count as a badge: "Filters (2)" when filters are applied
- Add a "Clear All Filters" button that appears when any filter is active

## 4.4 Transfer & Adjustment Entries
- Transfer entries (like "Transfer to IDFC") are shown with green text and "⇋ ₹2,00,000.00" which is good visual differentiation
- "Balance Adjustment" entries look like regular transactions but they're manual corrections — give them a distinct visual treatment:
  - Use a dashed border or muted/italic styling to differentiate from real transactions
  - Add a small info tooltip: "Manual balance adjustment — not an actual transaction"

## 4.5 Category Badges
- The colored category badges are visually clear and work well
- Consider adding a click interaction: clicking a category badge should filter the table to show only transactions in that category (acting as a quick filter)

## 4.6 Amount Column
- Right-aligned amounts are correct
- Add visual differentiation for income vs expense amounts:
  - Expenses: show in default white/light text (current behavior)
  - Income: show in green with a "+" prefix
  - Transfers: keep the current green with "⇋" prefix
- Consider adding a running balance column (optional, togglable) showing the account balance after each transaction
