Objective: Update the SurplusEngine Finance App to support multiple bank accounts, credit cards, inter-account transfers, and a custom billing cycle that resets on the 25th of every month.

1. Database & Schema Updates (Drizzle/PostgreSQL):

New Tables: * accounts: (id, name, type [Bank/Credit Card], current_balance, credit_limit [for CCs]).

categories: (id, name, type [Income/Expense]). Seed this with the initial Excel categories but allow CRUD.

Transaction Table Update: Add accountId (source) and toAccountId (destination for transfers). Add 'Transfer' as a valid transaction type.

Migration: Ensure all existing transactions are mapped to a default "Primary Bank" account during the transition.

2. Custom Billing Cycle Logic (The "25th" Rule):

Implement a global logic where "Current Month" = 25th of previous month to 24th of current month.

The Dashboard summary (Income, Expenses, Surplus) must reflect this cycle.

The "History" view must remain a continuous stream, but with a "Cycle Filter" (e.g., "Feb 25 - Mar 24").

3. Account & Transfer Functionality:

Transactional Impact: Logging an Expense must decrement the selected Account balance. Logging Income must increment it.

Transfer Logic: Implement a POST /transfers endpoint. A transfer between Bank A and Bank B must update both balances but must not appear as an Expense or Income in the Budget/Waterfall analysis.

CC Payments: Paying a Credit Card from a Bank account is a Transfer (Bank Balance decreases, CC Outstanding decreases).

4. New UI Pages & Components:

Manage Accounts Page (/accounts): A dashboard for all bank accounts and cards. Show a "Net Worth" card at the top (Total Bank - Total CC).

Category Manager: A settings view to add or delete spend categories. Update the transaction form to use this dynamic list.

Trend Charts (Recharts):

CC Spend Trend: A line chart showing cumulative spending in the current cycle.

Living Expenses Trend: A bar chart comparing the "Living Expenses" category across the last 6 billing cycles.

Transfer Modal: A simple UI to select 'From Account', 'To Account', Amount, and Date.

5. Responsive UI Refinement:

Maintain the "Mobile-First" approach: All new Account and Category lists must transform into Card Views on mobile and Data Tables on desktop.

Ensure the Transfer Modal is easy to use on a touch screen.
