## Setup Assumptions
- User has these accounts: HDFC Salary, Jupiter Savings, IDFC Savings, One Card, Axis Vistara, Axis Magnus, HDFC Tata Neu, HDFC Regalia Gold, AU Zenith Bank, Amex Travel Platinum
- User has these categories: Food, Drinks, Transportation, Shopping, Subscriptions, Father, Term Insurance, Health Insurance, SIP, Loan EMI, Spa, Electricity, Grocery, Medical, Personal, Gifts, Bonus, Interest, Salary, Cashback, Other
- Today's date: 2026-04-04
- User has existing transaction history (referenced in specific scenarios below)

## 5. EDGE CASES — Corrections and Cancellations

### 5.1 Amount correction mid-flow
**Input:** "200 on drinks"
**AI:** "₹200 on Drinks, today. Which account?"
**User:** "wait make it 350"
**Expected:** Updates amount to 350, re-asks: "₹350 on Drinks, today. Which account?"

### 5.2 Category correction mid-flow
**Input:** "500 on food from one card"
**AI:** Confirmation card — ₹500, Food, today, One Card.
**User:** "actually that's drinks not food"
**Expected:** Updates category to Drinks, shows updated confirmation card.

### 5.3 Correction at confirmation stage
**AI shows confirmation:** ₹200, Drinks, Starbucks, today, Axis Vistara.
**User:** "change account to one card"
**Expected:** Updates account, shows revised confirmation card.

### 5.4 Negation
**Input:** "500 on drinks, not from one card"
**Expected:** Removes One Card from suggestions when asking for account. Should not pre-select One Card even if it's the most frequent.

### 5.5 Cancel at any point
**User (mid-flow):** "cancel" / "never mind" / "forget it"
**Expected:** "No worries, cancelled." Clears the current flow. Input resets to idle state.

### 5.6 Cancel then start new
**User:** "cancel"
**AI:** "No worries, cancelled."
**User:** "200 food zomato"
**Expected:** Starts fresh. No leakage from the cancelled transaction.

## 8. DATE PARSING

### 8.1 Relative dates
- "yesterday" → Apr 3, 2026
- "day before yesterday" → Apr 2, 2026
- "last friday" → Mar 27, 2026 (or closest previous Friday)
- "last week" → should ask for specific date or default to start of last week

### 8.2 Explicit dates
- "on march 15" → Mar 15, 2026
- "on 1st april" → Apr 1, 2026
- "3rd" → Apr 3, 2026 (current month assumed)

### 8.3 No date
- Any input without a date → defaults to today (Apr 4, 2026). Never asks.

### 8.4 Future date warning
- "500 food tomorrow" → Should warn: "That's a future date (Apr 5). Log it anyway?" [Yes] [Change to today]

## 9. AMOUNT PARSING

### 9.1 Various formats
- "200" → ₹200.00
- "1.5k" → ₹1,500.00
- "2L" / "2 lakh" → ₹2,00,000.00
- "1500.50" → ₹1,500.50
- "2.5k" → ₹2,500.00

### 9.2 Amount with currency symbol
- "₹500" → ₹500.00
- "rs 300" / "Rs. 300" / "INR 300" → ₹300.00

### 9.3 No amount — hard failure
- "bought coffee at starbucks" → must ask "How much was it?"
- Should not proceed to any other slot until amount is provided.
